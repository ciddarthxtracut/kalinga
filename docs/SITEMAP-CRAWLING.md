# Sitemap and why online crawler tools only see some URLs

## What happens with “online sitemap generator” tools

Many **online sitemap generator tools** discover URLs by **crawling the site**: they start at the homepage and follow every link they find in the HTML. They do **not** necessarily request your real `sitemap.xml` first.

So they only list:

- **Static pages** that are linked from nav, footer, or other pages (e.g. `/about-us`, `/contact-us`, `/admissions`, …).

They **miss**:

- **Dynamic API-driven pages** that are not linked (or only loaded via JS):
  - `/departments/[slug]` (e.g. `/departments/faculty-of-law`)
  - `/courses/[slug]` (e.g. `/courses/mba`)
  - `/news-and-events/[slug]` (individual news/event pages)

Those URLs are not in the initial HTML of the homepage or main menus; they come from the API and are often only linked from listing pages (e.g. “All departments”, “All courses”). If the tool doesn’t execute JavaScript or doesn’t crawl those listing pages deeply, it never sees the links and doesn’t add those URLs.

## How we expose all URLs (including dynamic)

The **real sitemap** is generated by the app and includes both static and API-driven URLs:

- **URL:** `https://kalingauniversity.ac.in/sitemap.xml`
- **Contents:**
  - All static pages (same ~60+ you see from crawler tools)
  - All **department** pages from the API (`/departments/{slug}`)
  - All **course** pages from the API (`/courses/{slug}`)
  - All **news/events** pages from the API (`/news-and-events/{slug}`)

So the **canonical list of crawlable URLs** is in that file, not in what an online “crawl-only” tool finds.

## What we changed in code

1. **Revalidation**  
   The sitemap is revalidated every hour (`revalidate = 3600`). When someone (or Google) requests `sitemap.xml`, the app can refetch departments, courses, and news from the API and regenerate the sitemap. So the live sitemap stays in sync with the API.

2. **Logging**  
   On sitemap generation we log how many URLs were added (static vs departments vs courses vs news). If departments/courses/news are 0, the API may be failing at that moment (check server logs).

3. **Error handling**  
   If the departments, courses, or news API fails, we log a warning and still return the static entries so the sitemap is never empty.

## What you should use where

- **Google Search Console / Bing**  
  Submit: `https://kalingauniversity.ac.in/sitemap.xml`  
  That sitemap includes all static + dynamic URLs.

- **Online “sitemap generator” tools**  
  They will keep showing only the ~60 URLs they discover by crawling. To get the **full** list (including departments, courses, news), either:
  - Use the live sitemap: open `https://kalingauniversity.ac.in/sitemap.xml` and use that XML, or  
  - Use a tool that can **import** a sitemap URL (e.g. “Fetch sitemap from URL”) instead of crawl-only discovery.

- **Making crawlers discover more by links**  
  To help crawl-based tools (and users) find more pages, ensure:
  - There are server-rendered links to “All departments” and “All courses” (and ideally each department/course) from the main site (e.g. `/academics`, `/departments`).
  - The news/events listing page links to each `/news-and-events/[slug]` in the HTML so crawlers that follow links can discover them.
